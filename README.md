### Задание 1  Кэширование
Приведите примеры проблем, которые можно решить с помощью кэширования.



`Приведите ответ в свободной форме........`

Медленная загрузка страниц/сервисов
Когда приложение или сайт обращается к удалённой базе данных, внешнему API или выполняет сложные вычисления при каждом запросе, пользователи сталкиваются с задержками. Кэширование промежуточных результатов или готовых ответов позволяет отдавать данные почти мгновенно.

Высокая нагрузка на сервер и базу данных
При большом числе одновременных запросов сервер может не справляться с обработкой: растут время ответа, потребление CPU/памяти, возможны сбои. Кэширование снижает число «тяжёлых» операций (запросов к БД, сложных расчётов), разгружая инфраструктуру.

Дорогой или лимитированный внешний API
Если приложение регулярно обращается к платному API с жёсткими лимитами на число запросов, кэширование ответов позволяет:

сократить расходы (меньше вызовов = меньше плата);

избежать превышения лимитов и блокировок;

обеспечить работу при временных сбоях внешнего сервиса (отдавать закэшированные данные).

Повторяющиеся тяжёлые вычисления
Некоторые задачи (агрегация больших массивов, построение отчётов, рендеринг сложных графиков) требуют много ресурсов. Если результат меняется нечасто, его можно вычислить один раз, сохранить в кэш и отдавать повторно до следующего обновления.

Задержки сети (latency) при геораспределённых сервисах
Когда сервер и пользователи находятся на разных континентах, каждый запрос проходит большое расстояние. Распределённое кэширование (CDN, edge-кэши) позволяет хранить данные ближе к пользователям и снижать время ответа.

Частый доступ к редко изменяемым данным
Справочники, настройки, статический контент (изображения, CSS/JS) запрашиваются многократно, но обновляются редко. Кэширование на стороне клиента (браузера) или прокси-сервера избавляет от повторных загрузок.

Пиковые нагрузки (например, при релизах или рекламных акциях)
В моменты резкого роста трафика кэш «держит удар»: отдаёт популярные данные без обращения к основному хранилищу, предотвращая перегрузку системы.

Снижение стоимости инфраструктуры
За счёт уменьшения числа обращений к дорогим ресурсам (БД, микросервисам) можно использовать менее мощные сервера или сократить число инстансов, сэкономив на железе/облачных ресурсах.

Улучшение пользовательского опыта (UX)
Быстрые ответы, плавная работа интерфейса, отсутствие «подвисаний» напрямую влияют на удовлетворённость пользователей и показатели вроде времени на сайте/конверсии.

Резервирование данных при временных сбоях
Если основной сервис недоступен, кэш может отдавать актуальные на момент сбоя данные, обеспечивая непрерывность работы (с оговоркой на возможную неактуальность).
